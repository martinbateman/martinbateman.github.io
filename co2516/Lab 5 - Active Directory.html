Introduction

In this lab you are going to add a Windows 10 computer to your Windows Domain. To do this you are going to add a Computer account to the domain and then add some more user accounts.

    Joining the domain

  1. Change to your Windows 10 Virtual Machine
    2. [help]
  2. Make sure that you Windows 10 Virtual Machine uses the same virtual network (VMnet) as your Windows server
    3. [help]
  3. Set the IP address to 10.0.1.10. You should set appropiate netmask, DNS and default route
    4. [help]

    An appropiate network, dns and default route mean setting that would work on this network. You have already set them up for the server we used a netmask of 255.255.255.0 (in other words a /24).

    The DNS has to be set to the Windows Server. On the Server you set the DNS to 10.0.1.2 which is the Windows Server. The DNS has to be set to the Windows Server in order to join the Active Directory Domain. When we join the domain the first thing that will happen is that the Windows 10 computer will look up the AD domain name in the DNS. The easiest way to make this work is to use point it at the Windows Server which is hosting the DNS.

  22. Using the information from IT Geek Rambling use a Group Policy to set a specific desktop background for the users in your domain.
    23. [help]

    You have to make the change on the Windows Server but the image will show on the Windows client. This menas that the Windows client must have access to the image you decide to use as a background. The easiest way to make sure the Windows client can access the image is to put the image on the Windows client. The path you set for the policy will be the path on the Windows client and wouldn't work on any other machine.
