    Introduction

    The purpose of this exercise is to get comfortable with reconnaissance techniques and tools.

    You should make sure you are comfortable with the UNIX command line before starting this exercise.

    You should not target a host outside of the Lab for active probing as this could be perceived as an attack.

    Instructions
  1. You should search online for articles stating which systems the School or University has. Try to find out the following
    • How is Blackboard hosted. Can you determine anything about the technologies used?
    • Which email service do UCLan use?
    • Can you find any exploits for this provider? Do not attempt to use them.
    2.  
  2. Use either http://whois.domaintools.com/ or the whois command to gather information on the University's domain.
    3.  
  3. Use the RIPE database at http://www.db.ripe.net/whois to find IP ranges of the University.
    4.  
  4. Use dig/nslookup or http://www.kloth.net/services/dig.php to query the DNS to find useful IP addresses - look for MX (mail) and NS (name server)
    5.  
    Active machines

    At this point we are going to move into the virtual machines you have created in Azure Lab Services since you are going to be sending active probing which could look like an attack.

  5. Start Kali and youe CO4509 Webserver. Log into Kali Linux and find its IP address using "ip addr". Now using either ping or nmap perform a ping sweep on from 10.0.0.1 to 10.0.0.255.
    6.  
  6. Perform a port scan using nmap. What are the open services? What versions numbers are in use?
    7.  
  7. Use either nmap or telnet to attempt to identify the host operating system.
    8.  
  8. Use Google to find a list of exploits for the host - combination of OS and services running. Search for common services first (http, ssh, etc) as they are more likely to have listed updated.
    9.  
    Performing the attack

    At this point you would be in a position to perform a remote attack against the CO4509 Web Server

    What aspects of the network/hosts/applications could you change to hinder your information gathering?
