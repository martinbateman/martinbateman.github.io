Introduction

You will be creating your first simulation of a real system and you will use this simulation to answer a number of questions. This lab session will last two weeks.

In this practical you will simulate what happened during the first lab session. You should create the topology below, which represents a simplified version of the topology for the lab. There are 20 client machines each of which has to download a 100MB ISO file from the FTP server. All 20 clients will engage in browsing the web and five clients will watch streaming video during the lab session.

Everyone will start to download the ISO within the first five minutes of the lab session.

Everyone will start browsing the web within the first five minutes of the lab and will continue until the end of the lab.

The streaming videos will start randomly within the lab session and last a random amount of time, between 15 and 300 seconds. The lab will be 1 hour long.

You should simulate this topology and scenario and use your simulation to answer the following questions:

  • In the worst case how long will it take all client machines to download the ISO image needed for the lab?
  • What is the mean throughput used by streaming video on the ingress?
  • What is the mean throughput used for web traffic
    • on the egress link between the lab switch and the CEPS switch?
    • on the ingress link between the lab switch and the CEPS switch?
    • on the egress link between the CEPS switch and the external switch1?
    • on the ingress link between the CEPS switch and the external switch1?
Notes
  • Any nam file produces from this simulation as described will be too large to load into nam. You can however use nam to check the network topology before you run large amounts of traffic over it.
  • The FTP server uses Cubic as the TCP variant.
  • You don't need to specify any other TCP variant.
