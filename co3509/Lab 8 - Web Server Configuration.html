    Introduction

    In this lab you are going to configuring Apache on www1 and www2 and populate the www root with a mirror of a website. We are going to use a mirror of the UCLan website. In this week's lab you will populate the website on both web servers with content. At the moment each web server will be seperate and next week you will configuring a load balancer.

    Instructions
  1. First you should install the Apache web server on each of www1 and www2.

    mb@www1:~$ sudo apt-get update
    mb@www1:~$ sudo apt-get install apache2

    mb@www2:~$ sudo apt-get update
    mb@www2:~$ sudo apt-get install apache2

    2.  
  2. From your home directory on each of the web servers (www1 and www2) mirror a subset of the UCLan website using the wget command.

    mb@www1:~$ wget -mpk --level=1 http://www.uclan.ac.uk/

    mb@www2:~$ wget -mpk --level=1 http://www.uclan.ac.uk/

    This creates a www.uclan.ac.uk directory in your home directory that contains a copy of a subset of the UCLan website.
    3.  
  3. Deploy the website mirror into apache.

    mb@www1:~$ cp -R www.uclan.ac.uk/* /var/www/

    mb@www2:~$ cp -R www.uclan.ac.uk/* /var/www/

    4.  
  4. Test that you are able to access the content from the client using a web browser. Debug any problems and repeat the procedure for the second web server.

    mb@www1:~$ cp -R www.uclan.ac.uk/* /var/www/

    mb@www2:~$ cp -R www.uclan.ac.uk/* /var/www/

    5.
